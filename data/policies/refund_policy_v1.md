# 退款政策 v1

## 1. 政策目的

本政策用于支持售后 Agent 判断订单是否可以进入退款复核流程。它是 Demo 合成政策，仅用于项目演示，不代表任何真实平台政策。

## 2. 适用范围

本政策适用于普通商品订单的售后退款判断。退款判断应结合订单履约状态、支付状态、客户诉求、商品问题证据和平台风险规则。

当前 Olist 数据集不包含真实退款流水、客服会话和售后附件，因此本项目只能做“退款复核候选”判断，不能做真实退款执行。

## 3. 基础退款候选条件

订单状态为 `delivered` 且用户在签收后 7 个自然日内提出申请，可以进入退款资格判断。

订单状态为 `canceled` 或 `unavailable` 时，优先进入履约异常退款流程，由人工确认是否已退款、是否需要补发或是否需要联系卖家。

## 4. 质量问题复核

若用户明确描述商品破损、错发、严重质量问题，且订单评论分数小于等于 2 分，可以生成质量问题人工复核工单。

在当前 Olist 数据集中，可以用 `reviews.review_score` 作为弱信号；`review_comment_message` 只能作为辅助证据，不能单独决定退款。

## 5. 高风险退款

以下情况必须人工审批：

- 订单支付金额高于平台 P90 金额
- 同一客户近期存在多次退款
- 政策证据不足
- 需要创建工单、补偿建议或资金动作
- 客户文本中包含威胁、投诉升级、法律风险或媒体曝光风险

## 6. Agent 执行限制

Agent 可以输出“初步符合退款复核条件”“建议创建待审批工单”“证据不足需人工复核”等判断。

Agent 不得直接承诺退款成功，不得绕过人工审批创建资金动作，也不得把弱信号包装成确定结论。
